# Vision

## What We Are Building

A crowdsourced indoor positioning platform that lets users locate items, services, and points of interest inside buildings — without GPS.

The system does not require dedicated infrastructure (beacons, QR codes, fixed sensors). It learns from the crowd: every walk contributes to a shared map.

## Target Use Cases

- Find a specific product in a supermarket
- Find Gate B12 in an airport
- Find a shop in a shopping mall
- Find a conference room in an office building

## What We Are Not Building

Clue is not indoor GPS. It is:

1. A sensor collection system
2. A path reconstruction engine
3. A collaborative mapping layer
4. A fingerprinting database
5. An item discovery interface

Localization is one component among five, not the product.

## Success Criteria

### Version 0.1

A user can start recording, walk, stop, and upload a sensor session.

### Version 0.2

A user can browse and replay their recorded sessions.

### Version 0.3

The system clusters trajectories and detects walkable areas automatically.

### Version 0.4

Users can drop markers, add photos, and name locations.

### Version 0.5

The system estimates the user's current location within 2–5 meters and can guide them to a tagged item.

## Long-Term Vision

A building becomes navigable the moment a few users walk through it. No prior setup required. The more people walk, the more accurate the map becomes.
